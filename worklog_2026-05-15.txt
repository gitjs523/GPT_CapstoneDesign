2026-05-15 작업 요약

1. 인증 API 연동 확인 및 보강
- Swagger 기준 auth-controller의 signup, login, logout, refresh 스펙을 확인했다.
- signup은 POST /api/auth/signup, login은 POST /api/auth/login으로 연동되어 있음을 확인했다.
- login/signup 응답의 accessToken을 localStorage의 snow.accessToken에 저장하는 흐름을 확인했다.
- logout 요청에 Authorization: Bearer accessToken 헤더가 붙는 것을 브라우저 Network 탭에서 확인했다.
- refreshAccessToken(), authFetch() 공통 함수를 추가했다.
- authFetch()는 인증 요청에 Bearer 토큰을 자동으로 붙이고, 401 발생 시 /api/auth/refresh 호출 후 원래 요청을 한 번 재시도한다.

2. 노트북 API 연동
- 기존 localStorage 중심의 Notebook 저장 흐름에 서버 API를 연결했다.
- 페이지 진입 시 로그인 토큰이 있으면 GET /api/notebooks로 서버 노트북 목록을 동기화한다.
- 새 Notebook 저장 시 POST /api/notebooks를 호출한다.
- 기존 Notebook 저장 시 PATCH /api/notebooks/{notebookId}를 호출한다.
- Notebook 삭제 시 DELETE /api/notebooks/{notebookId}를 호출한다.
- 서버가 반환한 notebookId를 currentNotebookId로 저장해 문서 API에서 사용할 수 있게 했다.
- 로컬에만 있는 임시 Notebook 상태는 서버 동기화 중에도 사라지지 않게 보존했다.

3. 문서 업로드 API 연동
- notebook-document-controller 기준으로 문서 업로드 API를 연결했다.
- 파일 업로드 시 POST /api/notebooks/{notebookId}/documents를 multipart/form-data로 호출한다.
- FormData의 file 필드에 선택한 파일을 담아 전송한다.
- Content-Type은 직접 지정하지 않고 브라우저가 multipart boundary를 자동으로 넣게 했다.
- 업로드 응답의 documentId, notebookId, originalFileName, fileType, fileSize, pageCount, analysisStatus, uploadedAt을 프론트 상태에 저장한다.
- 파일 용량 문제로 ERR_CONNECTION_RESET이 발생할 수 있음을 확인했고, 용량 조정 후 업로드가 정상 동작함을 확인했다.

4. 문서 분석 상태 polling 구현
- 업로드 직후 documentId를 사용해 GET /api/notebooks/{notebookId}/documents/{documentId}/status를 polling하도록 구현했다.
- polling 간격은 3초, 최대 100회로 설정했다.
- analysisStatus가 COMPLETED이면 분석 완료로 처리한다.
- analysisStatus가 FAILED이면 프론트 문서 목록에서도 해당 문서를 제거한다.
- summaryText가 오면 문서 상태에 저장한다.
- 분석 진행 중에는 polling 횟수 기반으로 진행률을 계산해 최대 99%까지 표시한다.
- 완료 시에는 분석 완료 상태로 변경한다.

5. 문서 삭제 및 분석 중단 UI 처리
- 업로드된 문서 목록의 x 버튼 클릭 시 DELETE /api/notebooks/{notebookId}/documents/{documentId}를 호출하도록 연결했다.
- x 버튼을 누르면 해당 documentId의 status polling을 중단한다.
- 백엔드 DELETE가 분석 job cancel까지 처리하면 실제 분석도 중단될 수 있다는 점을 정리했다.
- UI에서는 x 클릭 후 문서를 즉시 사라지게 하지 않고, 기존 진행률 위치에 "문서 분석이 중단되었습니다." 문구를 표시하도록 변경했다.
- CANCELLED 상태 표시용 스타일을 추가했다.

6. 업로드 UI 정리
- 기존 상단의 전체 업로드/분석 상태 pill UI를 제거했다.
- 분석 진행률을 각 업로드 문서 파일명 옆에 표시하도록 변경했다.
- 문서별 상태 표시를 추가했다.
  - 분석 대기
  - 분석 중 n%
  - 분석 완료
  - 분석 실패
  - 문서 분석이 중단되었습니다.
- 섹션 제목을 "업로드된 문서 완료" 계열에서 "문서 목록"으로 변경했다.
- 제목 뒤에 자동으로 붙던 "완료" 문구를 제거했다.
