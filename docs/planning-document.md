# SNOW (Smart Note Organizer for Web)
## 1. 기획 이유

대학생들은 시험 준비 과정에서 강의 자료를 반복적으로 검토하고, 핵심 내용을 정리하거나 예상 문제를 직접 구성하며 학습 내용을 점검한다. 그러나 이러한 과정은 많은 시간과 노력을 요구하며, 강의 자료의 분량이 많은 경우 학습 효율이 저하될 수 있다.

본 프로젝트는 강의 자료를 기반으로 전체 요약, 특정 단원 요약, 시험문제 생성, 개념 설명 기능을 제공하는 웹 서비스를 구현함으로써, 사용자의 학습 효율을 향상시키는 것을 목적으로 한다.

---

## 2. 기획 목표

다양한 학습 문서를 기반으로 요약, 문제 생성, 정답 및 해설 제공 기능을 수행하는 AI 학습 지원 서비스이다.

### 1) 문서 전체 요약 기능

업로드된 강의 자료 전체를 분석하여 핵심 주제와 주요 내용을 자동으로 요약하는 기능이다.
이를 통해 사용자는 문서의 전체적인 흐름과 핵심 내용을 신속하게 파악할 수 있다.

### 2) 특정 소단원 요약 기능

사용자가 원하는 단원, 소주제 또는 키워드를 입력하면 해당 범위의 내용만 선택적으로 요약하는 기능이다.
이를 통해 필요한 범위에 한정하여 효율적으로 복습할 수 있도록 지원한다.

### 3) 시험문제 자동 생성 기능

강의 자료를 기반으로 객관식, 단답형, 서술형 등의 시험문제를 자동으로 생성하는 기능이다.
이를 통해 사용자는 학습 내용을 문제 형식으로 점검하고 시험 대비에 활용할 수 있다.

### 4) 정답 및 해설 제공 기능

생성된 시험문제에 대한 정답과 해설을 함께 제공하는 기능이다.
이를 통해 사용자는 문제 풀이 이후 개념을 재확인하고 학습 내용을 보완할 수 있다.

### 5) 개념 상세 설명 기능

사용자가 이해하기 어려운 문장이나 개념에 대해 추가 설명을 요청할 수 있는 기능이다.
이를 통해 기존 요약보다 더 쉬운 표현이나 상세한 설명을 제공하여 학습 이해도를 높일 수 있다.

### 6) 문서 기반 질의응답 기능

사용자가 텍스트 입력창에 질문을 입력하면, 업로드된 강의 자료를 기반으로 관련 답변을 생성하는 기능이다.
이를 통해 일반적인 응답이 아닌, 문서 내용을 근거로 한 질의응답을 제공할 수 있다.

---

## 3. 사용자 제공 기능

본 시스템에서 사용자는 다음과 같은 기능을 수행할 수 있다.

* **강의 자료 업로드**
  사용자는 PDF, PPT 등의 강의 자료를 웹에 업로드할 수 있다.

* **문서 전체 요약 확인**
  사용자는 문서 업로드 직후 시스템이 자동 생성한 전체 요약 결과를 확인할 수 있다.

* **특정 단원 요약 요청**
  사용자는 원하는 단원, 키워드, 소주제를 입력하여 해당 범위에 대한 요약을 요청할 수 있다.

* **시험문제 생성 요청**
  사용자는 특정 범위 또는 주제를 기준으로 시험문제 생성을 요청할 수 있다.

* **정답 및 해설 확인**
  사용자는 생성된 문제에 대한 정답과 해설을 확인할 수 있다.

* **추가 설명 요청**
  사용자는 요약 내용이나 문서 내용 중 이해되지 않는 부분에 대해 추가 설명을 요청할 수 있다.

* **문서 내용 기반 질의응답**
  사용자는 입력창에 자유롭게 질문을 입력하고, 업로드된 강의 자료를 기반으로 답변을 제공받을 수 있다.

## 4. 아키텍처 구조

본 시스템은 문서 처리, 임베딩 기반 검색, 생성 모델 응답, 웹 서비스 제공 구조로 구성된다.

### 4.1 사용 모델

#### 1) 임베딩 모델

임베딩 모델은 강의 자료를 벡터 형태로 변환하여 저장하는 데 사용된다.
또한 사용자 질문 역시 동일한 방식으로 벡터화하여, 질문과 의미적으로 유사한 문서 구간을 검색하는 역할을 수행한다.
모델 :
* `qwen3-embedding:0.6b`

#### 2) 생성 모델

생성 모델은 문서 전체 요약, 특정 단원 요약, 시험문제 생성, 정답 및 해설 작성, 추가 설명 제공 등의 기능을 담당한다.
검색된 문서 내용을 바탕으로 사용자에게 제공할 자연어 응답을 생성하는 역할을 수행한다.
모델 :
* `qwen3:4b-q4_K_M`

#### 3) Ollama

Ollama는 로컬 환경에서 임베딩 모델과 생성 모델을 실행하기 위한 런타임으로 활용된다.
외부 유료 API 대신 오픈소스 모델을 직접 구동함으로써 비용을 절감하고, 반복적인 실험과 개발이 가능하도록 한다.

---

### 4.2 시스템 처리 구조

시스템의 전체 처리 흐름은 다음과 같다.

1. 사용자가 노트북에 강의 자료를 업로드한다.
2. 백엔드 서버가 문서에서 텍스트를 추출하고, 이를 Section 단위로 분할한다.
3. 분할된 Section을 임베딩 모델을 통해 벡터화하고, 메타데이터와 함께 PostgreSQL(`pgvector`)에 저장한다.
4. 생성 모델이 문서 전체에 대한 요약을 자동으로 생성한다.
5. 사용자가 특정 요약, 시험문제 생성, 상세 설명 등의 요청을 입력한다.
6. 사용자 요청을 임베딩하여 관련 Section을 검색한다.
7. 검색된 문서 내용을 바탕으로 생성 모델이 최종 응답을 생성한다.
8. 생성 결과를 웹 화면에 출력한다.

---

### 4.3 개발 및 배포 구조

본 프로젝트는 React 기반 프론트엔드, Spring 기반 백엔드, PostgreSQL(`pgvector`) 기반 저장소, Ollama 기반 AI 실행 환경으로 구성한다.
현재 목표는 운영 환경 완성보다 개발과 통합 검증을 우선하는 구조를 잡는 것이다.

1) 프론트엔드

프론트엔드는 React를 기반으로 구현한다.
개발 단계에서는 로컬 환경에서 React dev server를 실행하여 UI를 개발하고 백엔드와 연동한다.
프론트엔드는 사용자가 노트북에 문서를 업로드하고, 전체 요약 결과를 확인하며, 추가 질문이나 시험문제 생성을 요청할 수 있는 웹 인터페이스를 제공한다.

2) 백엔드 서버

백엔드 서버는 Spring Boot를 기반으로 구현하며, AWS EC2에서 Docker Compose로 실행하는 것을 목표로 한다.
백엔드 서버는 문서 업로드 처리, 텍스트 추출, 문서 분할, 임베딩 요청, 관련 문서 검색, 생성 모델 호출, 결과 반환 등 전체 시스템의 핵심 로직을 담당한다.
또한 프론트엔드와 AI 모델 실행 환경 사이에서 요청과 응답을 중계하는 중심 역할을 수행한다.

3) AI 실행 환경

AI 실행 환경은 Ollama를 기반으로 구성한다.
임베딩 모델은 `qwen3-embedding:0.6b`, 생성 모델은 `qwen3:4b-q4_K_M`을 사용한다.
개발 단계에서는 팀원 노트북 또는 연구실 로컬 컴퓨터에서 Ollama를 실행하고, ngrok 기반 터널을 통해 AWS EC2의 Spring 백엔드가 해당 Ollama API에 접근할 수 있도록 한다.
즉, 개발 중에는 항상 켜져 있는 중앙 모델 서버 대신 팀원 장비를 임시 모델 호스트로 활용한다.
다만 이 구조는 개발 편의를 위한 임시 방식이며, 이후 배포 단계에서는 항상 켜져 있는 안정적인 Ollama 호스트 또는 이에 준하는 구조를 별도로 결정해야 한다.

4) 저장소

저장소는 업로드된 문서 원본, 분할된 텍스트, 메타데이터, 임베딩 결과 및 생성 결과를 저장하는 역할을 수행한다.
관계형 데이터베이스는 AWS RDS의 PostgreSQL을 사용하며, 벡터 검색은 별도 Chroma 서비스 대신 PostgreSQL의 `pgvector`를 이용하여 처리하는 방향으로 설계한다.
즉, 문서 메타데이터와 임베딩 저장을 한 저장소 계층에서 관리하여 개발과 운영 복잡도를 줄이는 것을 우선한다.

5) 개발 단계 구성 요약

사용자 웹 브라우저 → React 로컬 개발 서버 → Spring Boot 백엔드(EC2) → ngrok → Ollama(임베딩 모델 / 생성 모델)

그리고 백엔드 서버는 별도로 AWS RDS PostgreSQL(`pgvector`)과 통신하여 문서 메타데이터, Section, 임베딩, 생성 결과를 저장 및 조회한다.

6) 배포 목표 구성 요약

사용자 웹 브라우저 → 프론트엔드 → Spring Boot 백엔드(EC2, Docker Compose) → Ollama 호스트(향후 확정)  
Spring Boot 백엔드(EC2, Docker Compose) ↔ AWS RDS PostgreSQL(`pgvector`)

### 4.4 배포 구성 요약

개발 단계 기준:

**사용자 웹 브라우저 → React 로컬 개발 서버 → Spring Boot 백엔드(EC2) → ngrok → Ollama(임베딩 모델 / 생성 모델)**

**Spring Boot 백엔드(EC2) ↔ AWS RDS PostgreSQL(`pgvector`)**

배포 목표 기준:

**사용자 웹 브라우저 → 프론트엔드 → Spring Boot 백엔드(EC2, Docker Compose) → Ollama 호스트(향후 확정)**

**Spring Boot 백엔드(EC2, Docker Compose) ↔ AWS RDS PostgreSQL(`pgvector`)**

이와 같은 구조를 통해 본 프로젝트는 단순한 텍스트 생성 시스템이 아니라, 강의 자료 업로드를 기반으로 자동 요약, 질의응답, 시험문제 생성을 수행하는 **문서 기반 학습 지원 웹 시스템**으로 구현하고자 한다.
